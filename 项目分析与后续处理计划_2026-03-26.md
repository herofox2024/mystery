# 推理资讯周报项目分析与后续处理计划

日期：2026-03-26

## 项目目标

这是一个 Python 自动化周报项目，目标是：

- 抓取豆瓣推理相关新书
- 抓取图书资讯类 RSS
- 通过规则过滤和 AI 精筛保留更相关内容
- 生成 Markdown 和 HTML 周报
- 支持定时执行

当前主流程文件：

- `main.py`
- `report.py`
- `scrapers/douban.py`
- `scrapers/rss_feeds.py`
- `scrapers/ai_filter.py`
- `config.yaml`

## 当前结论

项目思路清晰，结构不大，适合继续维护。但当前目录下这份代码不是完整可运行状态，存在核心源码缺失和报告模板损坏的问题，不能直接作为稳定生产版本使用。

## 已确认的问题

### 1. 项目当前无法直接运行

执行：

```powershell
python main.py --once
```

报错：

```text
ModuleNotFoundError: No module named 'app_helpers'
```

原因：

- `main.py` 依赖 `app_helpers.py`
- 目录中不存在 `app_helpers.py`
- 仅存在 `__pycache__/app_helpers.cpython-313.pyc`

影响：

- 配置补全
- 配置校验
- 规则过滤
- 打分
- 历史去重

这些核心逻辑的源码当前都缺失，项目无法在新环境复现运行。

### 2. 报告模板文件已损坏

`report.py` 中存在以下问题：

- Jinja 变量写法异常，部分位置不是标准 `{{ ... }}`
- 部分 HTML 标签损坏
- 模板中混入残缺字符串
- 生成的 HTML 结构不稳定

实际输出文件中已经能看到：

- 中文显示异常
- 特殊字符打坏链接结构
- 个别标签闭合错误

### 3. 编码或文本内容存在异常

项目中多个文件显示出明显乱码特征，包括：

- `README.md`
- `main.py` 中的中文注释和日志
- `report.py` 中的中文模板文本
- `config.yaml` 中的中文配置值
- `data/state.json` 中的中文标题

需要进一步确认：

- 是文件本身编码已损坏
- 还是终端读取方式导致显示异常

不排除两者同时存在。

### 4. RSS 精度不足

当前 RSS 来源较泛，例如：

- The Guardian Books
- Literary Hub
- Book Riot

这些源本身并不只发布推理小说相关内容。当前输出结果中已经出现不少主题偏离的资讯，说明仅靠现有规则和 AI 精筛还不足以保证周报主题聚焦。

### 5. AI 过滤存在“看似开启、实际弱过滤”的风险

当前行为特征：

- 配置允许多个 provider 顺序回退
- 没有 API Key 时会跳过 AI
- AI 未返回有效决策时，条目常被默认保留

影响：

- `ai_filter.enabled: true` 不代表真的做了有效筛选
- 最终周报可能混入大量泛图书资讯

### 6. 当前目录不是 Git 仓库

执行 `git status --short` 返回：

```text
fatal: not a git repository
```

影响：

- 无法直接追踪变更
- 无法比对历史修复
- 不利于后续维护和回滚

## 项目优点

虽然当前状态不完整，但仍有一些设计值得保留：

- 主流程分层清晰，采集、过滤、去重、输出职责分开
- 豆瓣抓取器使用了 `requests.Session` 和重试机制
- AI 模块支持多 provider 顺序回退
- 输出同时支持 Markdown 和 HTML
- 有状态文件 `data/state.json`，说明设计上考虑了历史去重
